GitHub HW-2
1. На локальном репозитории сделать ветки для:
- Postman
$ git branch Postman

- Jmeter
$ git branch Jmeter

- CheckLists
$ git branch CheckLists

- Bug Reports
$ git branch Bug-Reports

- SQL
$ git branch SQL

- Charles
$ git branch Charles

- Mobile testing
$ git branch Mobile-Testing

2. Запушить все ветки на внешний репозиторий
$ git push origin --all

3. В ветке Bug-Reports сделать текстовый документ со структурой баг репорта
$ git switch Bug-Reports
$ vim bug-report-structure.txt

4. Запушить структуру багрепорта на внешний репозиторий
$ git add .
$ git commit -m "create and fill in bug-report file"
$ git push -u origin Bug-Reports

5. Вмержить ветку Bug-Reports в Main
$ git checkout main
$ git merge Bug-Reports

6. Запушить main на внешний репозиторий.
$ git push

7. В ветке CheckLists набросать структуру чек листа.
$ git switch CheckLists
$ vim Checklist-structure.txt

8. Запушить структуру на внешний репозиторий
$ git add .
$ git commit -m "create and fill in Checklist file"
$ git push -u origin CheckLists

9. На внешнем репозитории сделать Pull Request ветки CheckLists в main
Репозиторий Git-HW-2 ветка main --> Кнопка "Compare & Pull Request" --> Кнопка "Create pull request" --> Кнопка "Merge pull request" --> Кнопка "Confirm merge"

10. Синхронизировать Внешнюю и Локальную ветки Main
$ git checkout main
git pull origin main
